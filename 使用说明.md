# MJ批量生图引擎 v2.0 — JSON主题包驱动

## 快速开始

双击 `dist\MJ批量生图_引擎.exe`，启动时自动弹窗选择主题。

---

## 文件结构

```
MJ批量生图_引擎/
├── MJ批量生图_引擎.pyw      ← 核心引擎（不需要动它）
├── MJ批量生图_引擎.exe      ← 打包后的可执行文件
├── themes/
│   ├── 巨构主义.json        ← 原版 v1.0 种子包
│   ├── 极简主义.json        ← 示例种子包
│   └── 结构主义.json        ← 示例种子包
└── 使用说明.md
```

---

## 添加新风格 — 只需一个 JSON 文件

1. 在 `themes/` 目录下新建 `你的风格.json`
2. 参照下方字段说明填写
3. 重启引擎，选择该主题即可

### JSON 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 窗口标题 |
| `theme_keyword` | string | 自动选图时的风格关键词（如"极简感"） |
| `struct_presets` | list | 文生图「结构」下拉选项 |
| `light_presets` | list | 文生图「光线」下拉选项 |
| `mood_presets` | list | 文生图「氛围」下拉选项 |
| `style_suffix` | string | 自动拼接在所有文生图 prompt 末尾的风格尾缀 |
| `blend_default_prompt` | string | 图生图无额外描述时的默认 prompt |
| `meta_prompt_layers` | string | 多图分层模式的 Gemini 系统提示词（含 `{n}`/`{roles}`/`{theme_clause}` 占位符） |
| `meta_prompt_multi` | string | 多图普通模式的 Gemini 系统提示词（含 `{n}`/`{roles}` 占位符） |
| `meta_prompt_single` | string | 单图模式的 Gemini 系统提示词 |
| `layer_labels_4` | list[4] | 4图分层时每张图的角色描述 |
| `layer_labels_multi` | list[4] | 多图普通模式时每张图的角色描述 |

### 最简模板

```json
{
  "title": "MJ你的风格名 v2.0",
  "theme_keyword": "你的风格感",
  "struct_presets": [
    "结构描述1",
    "结构描述2"
  ],
  "light_presets": [
    "光线描述1",
    "光线描述2"
  ],
  "mood_presets": [
    "氛围描述1",
    "氛围描述2"
  ],
  "style_suffix": "摄影级图像，8K质感，a single lone figure for scale，无水印",
  "blend_default_prompt": "超写实电影级图像，8K，a single lone figure for scale",
  "meta_prompt_single": "你是Midjourney提示词专家，专注于「你的风格」...\n\n只输出英文提示词，直接开始写",
  "meta_prompt_multi": "你是Midjourney提示词专家...\n{roles}\n...",
  "meta_prompt_layers": "...",
  "layer_labels_4": [
    "image1：风格图",
    "image2：结构图",
    "image3：氛围场景图",
    "image4：氛围场景图2"
  ],
  "layer_labels_multi": [
    "image1：构图参考",
    "image2：光影参考",
    "image3：结构参考",
    "image4：材质参考"
  ]
}
```

## 打包为 exe

```bash
pip install pyinstaller pillow requests
pyinstaller MJ批量生图_引擎.spec
```

打包后 `dist\MJ批量生图_引擎.exe` 可独立运行，`themes\` 目录已内嵌到 exe 内部。

---

## 与 v1.0 的区别

| 项目 | v1.0 巨构种子包 | v2.0 引擎 |
|------|----------------|-----------|
| 风格数据 | 硬编码在 .pyw | JSON 文件，随时可换 |
| 新增风格 | 需改代码重新打包 | 只需写一个 JSON |
| 分发 | 单风格 exe | 一个 exe + 多个 JSON |
| 底层逻辑 | 同 | 同（完全兼容） |
